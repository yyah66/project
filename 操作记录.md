- train/val/test = 4138/517/517（后因损坏图片过滤变为 4094/510/514）

### 遇到的问题及修复

1. **DocVQA 字段格式不匹配**：`query` 是多语言 dict（`{'en': '...', 'de': '...'}`），`answer` 是 dict（`{'text': '...'}`）→ 修改 `prepare_public_datasets.py` 的 `coalesce()` 处理 dict 取 `en` 键，`extract_answer()` 处理 dict 取 `text` 键

2. **COCO 图片下载慢**：原脚本 `wget` 串行下载→写 `download_coco_images.py` 用 `ThreadPoolExecutor(max_workers=20)` 并发下载

3. **TextVQA 图片 GCS 403**：`storage.googleapis.com` 国内必须走代理→改用 HF datasets 下载整个 validation 集（7GB，走 hf-mirror），然后 `extract_textvqa_images.py` 从缓存提取所需的 1706 张

4. **image token/feature 数量不匹配**：`transformers 5.9.0` 与 Qwen2.5-VL 的 image token 展开有兼容问题，不同尺寸图片导致 mismatch → 在 processor 加载后加 `min_pixels=256*28*28, max_pixels=512*28*28` 约束所有图片统一分辨率

5. **TextVQA 有损坏图片**：`Image.verify()` 不够严格，漏过截断图片（`OSError: image file is truncated`）→ 改用 `Image.load()` 做完整验证，过滤掉 44 张损坏图片

6. **评测 regex 失效**：`processor.decode(skip_special_tokens=True)` 去掉了 `<|im_start|>` 标签，导致正则无法提取 assistant 回复 → 改为 `skip_special_tokens=False`

**最终方案**: 在本地 Windows 用 `pip install easyocr` + `Reader()` 下载模型 → 上传 `.pth` 文件到 AutoDL `/root/.EasyOCR/model/`。

需要的文件：
- `craft_mlt_25k.pth`（检测，83MB）
- `zh_sim_g2.pth`（中文识别，22MB）
- `english_g2.pth` 未下载（中文模型已含英文字符集）
