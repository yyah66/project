# 消融实验执行计划

> 日期：2026-06-09 | 基座：Qwen2.5-VL-7B-Instruct | 测试集：merged_test.jsonl (514 条)

## 实验目标

量化三个因子对 VQA 准确率的独立贡献和交互效应：

- **OCR 文本注入**（ON/OFF）：是否将 OCR 提取的文字拼入 prompt
- **证据提示词**（ON/OFF）：是否使用 JSON 格式 + 证据字段的结构化提示约束
- **模型微调**（LoRA/Base）：是否加载 LoRA adapter

## 实验矩阵（2 × 2 × 2 = 8 组）

| 组号 | OCR | 证据提示 | 模型 | 文件名 | 说明 |
|------|-----|----------|------|--------|------|
| 1 | ON | ON | LoRA | `group1_ocr_on_evi_on_lora` | 全开（当前最佳 67.70%） |
| 2 | ON | ON | Base | `group2_ocr_on_evi_on_base` | 仅去掉微调增益 |
| 3 | ON | OFF | LoRA | `group3_ocr_on_evi_off_lora` | 仅去掉证据约束 |
| 4 | ON | OFF | Base | `group4_ocr_on_evi_off_base` | 纯 OCR 辅助，无微调无证据 |
| 5 | OFF | ON | LoRA | `group5_ocr_off_evi_on_lora` | 仅靠视觉+证据提示 |
| 6 | OFF | ON | Base | `group6_ocr_off_evi_on_base` | 仅靠视觉+证据提示（无微调） |
| 7 | OFF | OFF | LoRA | `group7_ocr_off_evi_off_lora` | 极简：纯视觉+简单提示+微调 |
| 8 | OFF | OFF | Base | `group8_ocr_off_evi_off_base` | 底线：纯视觉+简单提示+基座 |

## 因子操作定义

### OCR 开关

| 状态 | 行为 |
|------|------|
| ON | 对每张测试图调用 `OCRService.extract()`（rapidocr 快速模式），将 OCR 文本拼入 user prompt 的 `ocr_block` |
| OFF | 使用已有的 `NoOpOCRService`（`src/ocr.py:41-54`），返回空 OCR 结果，prompt 中不含 OCR 文本 |

### 证据提示开关

| 状态 | 行为 |
|------|------|
| ON | 使用 `build_system_prompt(evidence_enabled=True)` + `build_user_prompt(evidence_enabled=True)`，包含 JSON 格式约束（answer/evidence/confidence/uncertainty）和证据引用要求 |
| OFF | 使用 `build_system_prompt(evidence_enabled=False)` + `build_user_prompt(evidence_enabled=False)`，简化为纯图文问答提示，无 JSON 约束 |

### 模型状态

| 状态 | 行为 |
|------|------|
| LoRA | 加载 `outputs/lora/merged/final_adapter/`，与现有 `--adapter` 模式一致 |
| Base | 仅加载 Qwen2.5-VL-7B-Instruct 基座，与现有 `--base-only` 模式一致 |

## 需要改动的代码

### 1. 修改 `scripts/evaluate.py`

新增两个命令行参数，使推理路径参数化：

```
新增参数：
  --no-ocr              关闭 OCR 文本注入（默认 OCR 开启）
  --no-evidence-prompt  关闭证据提示词约束（默认证据提示开启）
```

内部逻辑改动（约 30 行）：

- 参数解析区新增两个 `store_true` 参数
- OCR=ON 时导入 `OCRService`，对每张图调用 `extract()` 取 OCR 文本
- OCR=OFF 时使用 `NoOpOCRService`
- Evidence=ON 时调用 `build_system_prompt(True)` / `build_user_prompt(True)`
- Evidence=OFF 时调用 `build_system_prompt(False)` / `build_user_prompt(False)`
- 评测指标不变，仍用 Exact Match（大小写不敏感）

### 2. 新建 `scripts/run_ablation.sh`

一键运行 8 组实验，每组输出独立 JSON：

```bash
#!/bin/bash
# 消融实验一键运行脚本
# 用法: bash scripts/run_ablation.sh

MODEL="/root/autodl-tmp/models/qwen/Qwen2.5-VL-7B-Instruct"
ADAPTER="outputs/lora/merged/final_adapter"
TEST="data/training/chartqa/merged_test.jsonl"
OUTDIR="outputs/ablation"
mkdir -p "$OUTDIR"

echo "=== 消融实验：8 组 ==="

# 组1: OCR=ON  Evidence=ON  LoRA
echo "[1/8] OCR=ON Evidence=ON LoRA"
python3 scripts/evaluate.py --adapter "$ADAPTER" --test-jsonl "$TEST" \
  --save-results "$OUTDIR/group1_ocr_on_evi_on_lora.json"

# 组2: OCR=ON  Evidence=ON  Base
echo "[2/8] OCR=ON Evidence=ON Base"
python3 scripts/evaluate.py --base-only --test-jsonl "$TEST" \
  --save-results "$OUTDIR/group2_ocr_on_evi_on_base.json"

# 组3: OCR=ON  Evidence=OFF  LoRA
echo "[3/8] OCR=ON Evidence=OFF LoRA"
python3 scripts/evaluate.py --adapter "$ADAPTER" --test-jsonl "$TEST" \
  --no-evidence-prompt \
  --save-results "$OUTDIR/group3_ocr_on_evi_off_lora.json"

# 组4: OCR=ON  Evidence=OFF  Base
echo "[4/8] OCR=ON Evidence=OFF Base"
python3 scripts/evaluate.py --base-only --test-jsonl "$TEST" \
  --no-evidence-prompt \
  --save-results "$OUTDIR/group4_ocr_on_evi_off_base.json"

# 组5: OCR=OFF  Evidence=ON  LoRA
echo "[5/8] OCR=OFF Evidence=ON LoRA"
python3 scripts/evaluate.py --adapter "$ADAPTER" --test-jsonl "$TEST" \
  --no-ocr \
  --save-results "$OUTDIR/group5_ocr_off_evi_on_lora.json"

# 组6: OCR=OFF  Evidence=ON  Base
echo "[6/8] OCR=OFF Evidence=ON Base"
python3 scripts/evaluate.py --base-only --test-jsonl "$TEST" \
  --no-ocr \
  --save-results "$OUTDIR/group6_ocr_off_evi_on_base.json"

# 组7: OCR=OFF  Evidence=OFF  LoRA
echo "[7/8] OCR=OFF Evidence=OFF LoRA"
python3 scripts/evaluate.py --adapter "$ADAPTER" --test-jsonl "$TEST" \
  --no-ocr --no-evidence-prompt \
  --save-results "$OUTDIR/group7_ocr_off_evi_off_lora.json"

# 组8: OCR=OFF  Evidence=OFF  Base
echo "[8/8] OCR=OFF Evidence=OFF Base"
python3 scripts/evaluate.py --base-only --test-jsonl "$TEST" \
  --no-ocr --no-evidence-prompt \
  --save-results "$OUTDIR/group8_ocr_off_evi_off_base.json"

echo "=== 消融实验完成 ==="
```

## 预期分析维度

### 主效应

| 因子 | 对比方式 | 回答的问题 |
|------|----------|-----------|
| LoRA vs Base | avg(组1,3,5,7) − avg(组2,4,6,8) | 微调带来多少提升？ |
| OCR ON vs OFF | avg(组1,2,3,4) − avg(组5,6,7,8) | OCR 文本注入是否有帮助？ |
| Evidence ON vs OFF | avg(组1,2,5,6) − avg(组3,4,7,8) | 证据提示约束是否有帮助？ |

### 交互效应

- OCR × Model：OCR 在 Base 上帮助大还是 LoRA 上帮助大？
- Evidence × Model：证据提示对 Base 更有效还是对 LoRA 更有效？
- OCR × Evidence：OCR 和证据提示是互补还是冗余？

## 风险与注意事项

1. **OCR 后端依赖**：需确认已安装 `rapidocr-onnxruntime`，否则 OCR=ON 的 4 组会降级为空 OCR
2. **证据提示格式变化**：Evidence=ON 时 prompt 包含 JSON 格式要求，可能改变模型输出分布；`_parse_model_output` 已有兜底解析，但需关注 JSON 解析失败率
3. **显存**：8 组串行执行，单组加载一次模型，无需同时驻留多份
4. **破损图片**：evaluate.py 已内置损坏图片跳过逻辑，无需额外处理
5. **评测指标**：统一使用 Exact Match（大小写不敏感），与现有评测保持一致
